Project Summary

My cognitive neuroscience research work involves collaboration with researchers at the Department of Neurosurgery at the University of California, San Francisco studying the impairments of cognitive and sensory functions in brain tumor patients. We collect behavioral and intracranial EEG (through placing electrodes directly on the brain) data while patients perform various computerized cognitive tasks that are programmed in MATLAB.

The Problem

Research technicians who are responsible for collecting the longitudinal cognitive data from the patients do not have the programming background and the familiarity with MATLAB to be able to quickly run and troubleshoot computerized cognitive testing.

The Solution

In an effort to streamline this testing process, I designed and implemented a graphical user interface for running computerized cognitive tests to be used by research technicians. After many iterations of the toolbox based on user feedback, the interface allows testers to:

  • run cognitive task scripts through a button press
  • adjust the parameters of the cognitive task without needing to edit the task script
  • save testing notes
  • allow them to draw on an image of the brain to denote electrode placement
  • pull up previous testing information for a specific subject
  • use checkbox of all the testing procedures to keep track of progress

Define & Research

In order to understand the needs and identify the pain points of the testing process, I conducted interviews of 3 different clinical coordinators without any programming background and 2 cognitive researchers who were designing these cognitive tasks.

Design & Develop

Based on the information I gathered from the clinical coordinators and researchers, I started out by sketching up with a very low fidelity prototype on a Word document so I could hash out what information and functionality would be necessary and helpful for the graphical user interface (GUI) that I was developing. I actually tried using this prototype during a patient testing to see what parts of the prototype I was actually using, where it needed additional development, and which sections were unncessary. Taking this information, I then developed a functional GUI in MATLAB.

Testing & Evaluation

Once I was done with this first iteration of the prototype, I gave a brief introduction of this GUI to 2 clinical coordinators who would be be the primary users of the toolbox during patient testing. Since most of the functionality in the GUI was self-explanatory, I then observed the two testers use the interface as if they were testing a patient. I noted their interactions as they went through the different functionality of the program and they provided feedback as to other functionalities that they would like added to this program.

This development process went through many iterative processes, where the clinical coordinators would provided new feedback and issues that they ran into while using this interface to test patients. Features that were not being used by the clinical coordinators were removed and more detailed features that were often being utilized during testing was implemented.
